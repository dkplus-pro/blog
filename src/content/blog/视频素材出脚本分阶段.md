# Vlog 视频理解与故事脚本生成项目分阶段执行方案

## 项目目标

用 Python 实现一个命令行项目，输入 vlog 视频，自动完成：

```text
视频预处理
音频转文字
视频片段切分
关键帧抽取
画面理解
片段总结
故事规划
最终剪辑脚本生成
```

最终运行：

```bash
python main.py analyze ./videos/input.mp4
```

最终输出：

```text
outputs/
  input/
    audio.wav
    transcript.json
    clips.json
    frames/
    vision_results.json
    clip_summaries.json
    story_plan.json
    final_script.md
    final_script.json
```

# 阶段 1：视频预处理

## 目标

实现视频基础信息读取、音频提取、输出目录创建。

## 丢给 Cursor 的提示词

```text
请实现阶段 1：视频预处理功能。

当前项目已经有基础目录结构，请只修改与视频预处理相关的文件。

目标：
1. 输入一个视频路径
2. 检查视频文件是否存在
3. 使用 ffmpeg/ffprobe 获取视频基础信息
4. 提取音频为 audio.wav
5. 为当前视频创建 outputs/{video_name}/ 输出目录
6. 输出 video_info.json

需要实现：
- app/utils/ffmpeg_utils.py
- app/services/video_preprocess_service.py
- app/models/video_models.py
- app/pipeline/analyze_pipeline.py 里调用预处理流程
- main.py 中 analyze 命令能触发该流程

输出文件：
outputs/{video_name}/audio.wav
outputs/{video_name}/video_info.json

video_info.json 格式类似：
{
  "video_path": "...",
  "output_dir": "...",
  "audio_path": "...",
  "duration": 120.5,
  "width": 1920,
  "height": 1080,
  "fps": 30
}

要求：
1. 使用 subprocess 调用 ffmpeg/ffprobe
2. 命令执行失败要给出清晰错误
3. 不要实现 ASR、抽帧、LLM 功能
4. 保持最小改动
5. README 补充 ffmpeg 安装说明
```

## 阶段验收

执行：

```bash
python main.py analyze ./videos/input.mp4
```

应生成：

```text
outputs/input/audio.wav
outputs/input/video_info.json
```

# 阶段 3：视频片段切分

## 目标

```text

先不要接 PySceneDetect，第一版只做固定时长切分。

要求：
1. 根据 video_info.json 的 duration 切分片段
2. 默认每 5 秒一个片段
3. 支持 .env 配置 CLIP_SECONDS
4. 支持命令参数 --clip-seconds 覆盖
5. 将 transcript.json 中与片段时间重叠的文字合并到 clip.asr_text
6. 输出 clips.json

需要修改：
- app/models/clip_models.py
- app/services/scene_detect_service.py
- app/utils/time_utils.py
- app/pipeline/analyze_pipeline.py
- main.py

clips.json 格式：
[
  {
    "clip_id": "clip_001",
    "start": 0.0,
    "end": 5.0,
    "asr_text": "今天我们临时决定去海边。"
  }
]

要求：
1. clip_id 从 clip_001 开始递增
2. 最后一段不足 5 秒也要保留
3. 如果 clips.json 已存在，默认跳过
4. 支持 --force 重跑
5. 不要实现抽帧和大模型功能
```

```text
请实现阶段 5：LLM Provider 抽象。

目标：
封装一个通用的 OpenAI-compatible 文本大模型调用类，后续用于片段总结、故事规划和脚本生成。

   LLM_BASE_URL

要求：
```

# 阶段 7：片段结构化总结

## 目标

把每个片段的 ASR + 画面描述合并，让文本大模型总结成“事件、情绪、故事角色、可用价值”。

## 丢给 Cursor 的提示词

```text
请实现阶段 7：片段结构化总结功能。

目标：
读取 clips.json 和 vision_results.json，把每个片段总结成可用于故事编排的结构化数据。

输入：
- clip_id
- start
- end
- asr_text
- visual_summary
- mood
- actions
- location

调用 LLMProvider，输出 clip_summaries.json。

每个片段输出格式：
{
  "clip_id": "clip_001",
  "start": 0.0,
  "end": 5.0,
  "event": "主角和朋友在车站赶车",
  "emotion": "紧张",
  "story_role": "conflict",
  "story_value": 8,
  "usable": true,
  "reason": "这个片段有明确动作和冲突，适合作为故事开头钩子。"
}

story_role 只能是：
opening
setup
conflict
turning_point
highlight
reflection
ending
b_roll

需要修改：
- app/prompts/clip_summary_prompt.py
- app/services/clip_summary_service.py
- app/models/clip_models.py
- app/pipeline/analyze_pipeline.py
- app/utils/json_utils.py

要求：
1. 让模型输出 JSON
2. 要做 JSON 解析失败兜底
3. 单个片段失败不能中断全流程，记录 error 字段
4. 如果 clip_summaries.json 已存在，默认跳过
5. 支持 --force 重跑
6. 不要实现故事规划和最终脚本
```

```text
请只修改：
- story_plan_prompt.py
- final_script_prompt.py
- 必要的数据字段

```
