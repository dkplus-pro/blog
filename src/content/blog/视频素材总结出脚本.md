# Vlog 视频素材理解与故事脚本生成技术方案

## 1. 项目目标

实现一个 Python 项目，通过命令行脚本运行，对 vlog 视频素材进行自动分析，最终输出一份可用于剪辑的故事脚本。

核心能力：

```text
输入：一段或多段 vlog 视频
处理：视频拆解、音频转文字、画面理解、片段总结、故事编排
输出：视频总结、素材时间线、故事结构、剪辑脚本、旁白文案
```

最终运行方式示例：

```bash
python main.py analyze ./videos/input.mp4
```

输出结果：

```text
outputs/
  input/
    transcript.json
    scenes.json
    frames/
    clip_summaries.json
    story_plan.json
    final_script.md
    final_script.json
```

# 2. 整体处理流程

推荐采用流水线架构：

```text
视频文件
  ↓
视频预处理
  ↓
音频转文字
  ↓
镜头/片段切分
  ↓
关键帧抽取
  ↓
画面理解
  ↓
片段结构化总结
  ↓
故事主题规划
  ↓
生成剪辑脚本
```

对应流程：

```text
1. 提取音频
2. ASR 转文字
3. 检测镜头变化
4. 为每个片段抽关键帧
5. 使用视觉模型理解画面
6. 合并 ASR + 画面描述
7. 大模型总结片段
8. 大模型规划故事线
9. 大模型输出最终剪辑脚本
```

# 3. 技术选型

## 3.1 视频处理

使用：

```text
ffmpeg
```

用途：

```text
提取音频
抽取关键帧
获取视频时长
截图
转码
```

Python 可以用：

```text
subprocess 调用 ffmpeg
```

不建议一开始用复杂视频库，ffmpeg 最稳定。

---

## 3.2 音频转文字

推荐：

```text
faster-whisper
```

原因：

```text
本地运行
成本低
中文识别效果可用
速度比原版 whisper 更好
适合批量处理 vlog
```

模型建议：

| 场景    | 模型         |
| ----- | ---------- |
| 速度优先  | `small`    |
| 效果均衡  | `medium`   |
| 准确率优先 | `large-v3` |

MVP 阶段推荐：

```text
medium
```

---

## 3.3 镜头切分

推荐：

```text
PySceneDetect
```

用途：

```text
检测视频镜头变化
把长视频拆成多个片段
```

如果第一版想更简单，也可以先按固定时长切：

```text
每 5 秒 / 10 秒切一个片段
```

建议第一版：

```text
固定 5 秒切片 + ASR 时间对齐
```

第二版再接 PySceneDetect。

---

## 3.4 画面理解

画面理解有两种方案。

### 方案 A：经济型 API

只抽关键帧，调用视觉大模型理解图片。

每个片段只取：

```text
1~3 张关键帧
```

不要整段视频丢给模型。

适合：

```text
成本可控
实现简单
效果稳定
```

### 方案 B：本地小模型

可以接本地多模态模型，比如通过：

```text
Ollama
LM Studio
本地 Qwen-VL / MiniCPM-V
```

适合：

```text
有显卡
希望降低 API 成本
能接受部署复杂度
```

第一版建议：

```text
先用 API 跑通，再抽象成 VisionProvider，后面可替换成本地模型。
```

---

## 3.5 文本大模型

用于：

```text
片段总结
故事规划
脚本生成
旁白润色
```

经济选择：

```text
DeepSeek V4 Flash / Pro
GLM
Qwen
硅基流动上的便宜模型
```

建议：

```text
普通总结用便宜模型
最终脚本润色用稍强模型
```

```json
}
```

负责每个片段抽帧。

策略：



## 5.5 `vision_understanding_service.py`

负责画面理解。

输入：

```text

关键帧图片
该时间段 ASR 文本
```

输出：

```json
{
  "clip_id": "clip_001",
  "visual_summary": "两个人拖着行李箱在车站奔跑，画面比较慌乱。",
  "location": "车站",
  "people": ["主角", "朋友"],
  "actions": ["奔跑", "拖行李", "赶车"],
  "objects": ["行李箱", "车站标识"],
  "mood": "紧张、匆忙"
}
```

注意：

```text
视觉模型只描述真实画面，不允许编造剧情。
```

## 5.6 `clip_summary_service.py`

负责把画面 + 文本合并成片段总结。

输入：

```json
{

  "end": 5,
  "asr_text": "我们快赶不上车了。",
```

输出：

```json
{
  "clip_id": "clip_001",
  "start": 0,
  "end": 5,
```

```md
一次计划外的海边旅行，从慌乱出发到最终释然。

## 剪辑脚本

| 时间 | 片段 | 画面 | 旁白 | 字幕 | 音乐/音效 | 剪辑建议 |
```

# 6. 数据模型设计

可以用 `pydantic` 定义结构，保证数据稳定。

## Clip 片段模型

```python
from pydantic import BaseModel
from typing import List, Optional

class Clip(BaseModel):
    clip_id: str
    start: float
    end: float
    asr_text: str = ''
    frame_paths: List[str] = []
    visual_summary: Optional[str] = None
    event: Optional[str] = None
    emotion: Optional[str] = None
    story_role: Optional[str] = None
    story_value: int = 0
    usable: bool = True
```

## StoryPlan 模型

```python
class StoryPlan(BaseModel):
    theme: str
    main_character: str
    emotional_arc: str
    core_conflict: str
    ending_message: str
    selected_clips: List[str]
```

## ScriptItem 模型

```python
class ScriptItem(BaseModel):
    timeline: str
    clip_id: str
    visual: str
    voiceover: str
    subtitle: str
    music: str
    editing: str
```

# 9. Prompt 设计

## 9.1 片段总结 Prompt

```text
你是一个专业 vlog 素材分析师。

请根据下面的片段信息，提取这个片段真实发生的内容。
不要编造没有出现的剧情。

输入信息：
- 时间段：{start} - {end}
- 语音文字：{asr_text}
- 画面描述：{visual_summary}

请输出 JSON：
{
  "event": "这个片段发生了什么",
  "emotion": "情绪",
  "story_role": "opening/setup/conflict/turning_point/highlight/reflection/ending/b_roll",
  "story_value": 1-10,
  "usable": true/false,
  "reason": "为什么这个片段适合或不适合放进故事"
}
```

---

## 9.2 故事规划 Prompt

```text
你是一个短视频编导。

下面是 vlog 的素材片段列表。
请基于真实素材，设计一个有起承转合的故事。
可以调整叙事顺序，但不能编造素材中不存在的事件。

要求：
1. 找出故事主题
2. 找出情绪变化
3. 找出主要冲突或看点
4. 选择最适合使用的片段
5. 输出 JSON

片段列表：
{clip_summaries}
```

---

## 9.3 最终脚本 Prompt

```text
你是一个专业短视频剪辑脚本师。

请根据故事规划和素材片段，生成一份可执行剪辑脚本。

要求：
1. 输出表格
2. 每一行包含：时间段、clip_id、画面、旁白、字幕、音乐/音效、剪辑建议
3. 旁白要自然，有故事感，不要像 AI 总结
4. 保持 vlog 真实感
5. 不要虚构素材里没有的画面
6. 前 3 秒必须有钩子

故事规划：
{story_plan}

素材片段：
{selected_clips}
```

# 10. 成本控制策略

重点策略：

```text
本地处理能做的都本地做
大模型只处理结构化结果
视觉模型只看关键帧，不看完整视频
强模型只用于最终脚本润色
```

具体做法：

## 10.1 本地处理

```text
ffmpeg：本地
faster-whisper：本地
镜头切分：本地
抽帧：本地
```

## 10.2 视觉模型省钱

不要：

```text
每秒都调用视觉模型
```

建议：

```text
每个片段 1 张关键帧
高价值片段再补 2 张
```

## 10.3 文本模型省钱

分层使用：

| 环节   | 模型   |
| ---- | ---- |
| 片段总结 | 便宜模型 |
| 故事规划 | 中等模型 |
| 最终润色 | 强模型  |
| 批量重试 | 便宜模型 |

# 16. 方案总结

这个项目的核心不是“直接理解视频”，而是把视频拆成可控的结构化数据：

```text
视频 → 时间线 → 片段 → 事件 → 情绪 → 故事角色 → 剪辑脚本
```

最经济、最稳的实现方式是：

```text
本地：ffmpeg + faster-whisper + 抽帧 + 切片
API：关键帧理解 + 片段总结 + 故事生成
输出：Markdown + JSON 脚本
```

第一版先做成命令行工具，不做 Web，不做自动剪辑。等脚本质量稳定后，再继续接自动剪辑和前端界面。
