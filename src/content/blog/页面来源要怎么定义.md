建议不要所有“来源”都叫 `source`，最好按语义拆开。否则后面会出现：

```ts
source = 'home'
source = 'share'
source = 'banner'
source = 'wechat'
source = 'task'
```

时间久了会很混乱。

---

# 1. 常见 query 来源参数命名

## 推荐优先级

| 场景      | 推荐 query 名                    | 示例                            | 说明                  |
| ------- | ----------------------------- | ----------------------------- | ------------------- |
| 从哪个页面跳来 | `from` / `fromPage`           | `?from=home`                  | 最常用                 |
| 功能入口来源  | `entry` / `entrySource`       | `?entry=chatTool`             | 表示从哪个业务入口进入         |
| 渠道来源    | `channel`                     | `?channel=share`              | 分享、广告、活动等渠道         |
| 营销投放来源  | `utmSource`                   | `?utmSource=wechat`           | 偏运营/投放              |
| 分享来源    | `shareFrom`                   | `?shareFrom=userCard`         | 专门表示分享入口            |
| 场景来源    | `sceneType` / `businessScene` | `?sceneType=invite`           | 业务场景，不建议直接叫 `scene` |
| 回跳地址    | `redirect` / `redirectUrl`    | `?redirect=/pages/home/index` | 不是来源，是跳转目标          |
| 上一个路由   | `referrer` / `referrerPage`   | `?referrer=orderDetail`       | 比 `from` 更偏“上一页”    |

---

# 2. 我最推荐的命名方案

对你这种 **uniapp / 小程序项目**，我建议分三类：

## 页面来源：用 `from`

适合判断“从哪个页面来”。

```ts
/pages/member/index?from=chat
/pages/member/index?from=home
/pages/member/index?from=profile
```

常量：

```ts
export const PAGE_FROM = {
  HOME: 'home',
  CHAT: 'chat',
  PROFILE: 'profile',
  SHARE: 'share',
} as const
```

---

## 功能入口：用 `entry`

适合判断“从哪个功能入口进入”。

```ts
/pages/create/index?entry=aiMusic
/pages/create/index?entry=aiVideo
/pages/create/index?entry=toolPanel
```

常量：

```ts
export const ENTRY_SOURCE = {
  AI_MUSIC: 'aiMusic',
  AI_VIDEO: 'aiVideo',
  TOOL_PANEL: 'toolPanel',
  TEMPLATE_CARD: 'templateCard',
} as const
```

---

## 渠道来源：用 `channel`

适合分享、活动、广告、二维码等。

```ts
/pages/index/index?channel=share
/pages/index/index?channel=qrcode
/pages/index/index?channel=activity
```

常量：

```ts
export const CHANNEL_SOURCE = {
  SHARE: 'share',
  QRCODE: 'qrcode',
  ACTIVITY: 'activity',
  PUSH: 'push',
} as const
```

---

# 3. 不太推荐的命名

## 不推荐所有都叫 `source`

```ts
?source=home
?source=share
?source=banner
?source=aiMusic
```

问题是它太泛了，后面你不知道它表示：

```txt
页面来源？
功能入口？
渠道？
活动？
分享？
```

如果一定要用 `source`，建议加前缀：

```ts
pageSource
entrySource
channelSource
shareSource
```

---

## 不推荐用 `type`

```ts
?type=home
```

`type` 太泛，通常应该留给业务类型，比如：

```ts
messageType
cardType
questionType
orderType
```

---

## 小程序里不建议自定义直接叫 `scene`

微信小程序本身有启动场景值 `scene` 的概念，所以业务 query 里建议避免直接叫：

```ts
?scene=xxx
```

更推荐：

```ts
?sceneType=invite
?businessScene=memberExpire
```

---

# 4. 常量文件名应该叫什么？

看你的项目规模。

## 简单项目推荐

```txt
src/constants/source.ts
```

里面放所有来源相关常量。

```ts
// src/constants/source.ts

export const PAGE_FROM = {
  HOME: 'home',
  CHAT: 'chat',
  PROFILE: 'profile',
  SHARE: 'share',
} as const

export const ENTRY_SOURCE = {
  AI_MUSIC: 'aiMusic',
  AI_VIDEO: 'aiVideo',
  TOOL_PANEL: 'toolPanel',
} as const

export const CHANNEL_SOURCE = {
  SHARE: 'share',
  QRCODE: 'qrcode',
  ACTIVITY: 'activity',
  PUSH: 'push',
} as const
```

---

## 更推荐的命名

如果这些参数主要用于页面跳转 query，我更推荐：

```txt
src/constants/route-source.ts
```

或者：

```txt
src/constants/page-source.ts
```

我个人更建议用：

```txt
src/constants/route-source.ts
```

因为它表达得更清楚：**这些来源参数是给路由 query 用的。**

---

# 5. 推荐目录结构

```txt
src/
  constants/
    route-source.ts
    route-query.ts
    route-path.ts
```

职责可以这样分：

```txt
route-path.ts    页面路径常量
route-query.ts   query key 常量
route-source.ts  query 来源值常量
```

例如：

```ts
// src/constants/route-query.ts

export const ROUTE_QUERY_KEY = {
  FROM: 'from',
  ENTRY: 'entry',
  CHANNEL: 'channel',
  REDIRECT: 'redirect',
} as const
```

```ts
// src/constants/route-source.ts

export const PAGE_FROM = {
  HOME: 'home',
  CHAT: 'chat',
  PROFILE: 'profile',
} as const

export const ENTRY_SOURCE = {
  AI_MUSIC: 'aiMusic',
  AI_VIDEO: 'aiVideo',
  TOOL_PANEL: 'toolPanel',
} as const

export const CHANNEL_SOURCE = {
  SHARE: 'share',
  QRCODE: 'qrcode',
  ACTIVITY: 'activity',
} as const
```

---

# 6. 类型也可以一起导出

```ts
export type PageFrom = (typeof PAGE_FROM)[keyof typeof PAGE_FROM]

export type EntrySource = (typeof ENTRY_SOURCE)[keyof typeof ENTRY_SOURCE]

export type ChannelSource = (typeof CHANNEL_SOURCE)[keyof typeof CHANNEL_SOURCE]
```

使用时：

```ts
function goMemberPage(from: PageFrom) {
  uni.navigateTo({
    url: `/pages/member/index?from=${from}`,
  })
}
```

调用：

```ts
goMemberPage(PAGE_FROM.CHAT)
```

---

# 7. 我的最终建议

你可以这样定：

```txt
query 参数名：
- from：页面来源
- entry：功能入口
- channel：渠道来源
- redirect：回跳地址
- businessScene：业务场景

常量文件名：
- src/constants/route-query.ts
- src/constants/route-source.ts
```

最推荐：

```txt
src/constants/route-source.ts
```

不要叫：

```txt
source.ts
common.ts
constant.ts
config.ts
```

这些太泛，后面会越来越乱。
