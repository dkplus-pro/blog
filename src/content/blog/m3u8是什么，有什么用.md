**m3u8 本质上不是视频文件，而是 HLS 的播放清单文件。**

你可以这样理解：

```text
MP4：一个完整视频文件
m3u8：一个播放目录，里面记录很多视频小片段的地址
```

HLS 是 Apple 提出的 HTTP Live Streaming 技术，用普通 HTTP/CDN 分发视频，并能根据网络情况动态调整播放质量。Apple 官方也说明，HLS 适用于点播和直播，并会根据网络条件优化播放。([Apple Developer][1])

---

## 1. m3u8 是什么？

一个普通的 `.m3u8` 文件里面可能长这样：

```m3u8
#EXTM3U
#EXT-X-TARGETDURATION:6
#EXT-X-VERSION:3
#EXT-X-MEDIA-SEQUENCE:0

#EXTINF:6.0,
segment_000.ts
#EXTINF:6.0,
segment_001.ts
#EXTINF:6.0,
segment_002.ts
#EXT-X-ENDLIST
```

它的意思是：

```text
播放 segment_000.ts
再播放 segment_001.ts
再播放 segment_002.ts
……
```

所以 m3u8 只是清单，真正的视频内容在这些 `.ts` 或 `.m4s / fmp4` 分片里。Apple 文档里也明确提到，HLS 会把音视频作为一系列小文件发送，常见片段时长大约 6 秒。([Apple Developer][2])

---

## 2. m3u8 / HLS 有什么用？

主要有几个用处。

### 适合长视频

比如几十分钟、一两个小时的视频。
如果直接用 MP4，用户虽然不一定要下载完整文件才能播，但长视频拖动、缓冲、网络波动处理会更依赖浏览器和服务器对 Range 请求的支持。

HLS 把视频切成很多小片段，播放器按需加载，更适合长视频播放和 CDN 分发。

---

### 支持自适应码率

这是 HLS 最大的价值。

比如你可以准备三套清晰度：

```text
360p
720p
1080p
```

然后 master.m3u8 里记录不同清晰度地址：

```m3u8
#EXTM3U
#EXT-X-STREAM-INF:BANDWIDTH=800000,RESOLUTION=640x360
360p/index.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=2500000,RESOLUTION=1280x720
720p/index.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=5000000,RESOLUTION=1920x1080
1080p/index.m3u8
```

播放器会根据用户网速、缓冲情况、设备性能，自动切换清晰度。

网好就播 1080p，网差就降到 360p，尽量减少卡顿。

---

### 适合直播

直播场景下，m3u8 会不断更新，里面持续追加新的视频片段。
MP4 更偏完整文件，天然不适合直播。

---

### 适合 CDN

HLS 是一堆普通 HTTP 文件：

```text
index.m3u8
segment_000.ts
segment_001.ts
segment_002.ts
```

所以可以直接放 CDN、COS、OSS、Nginx 静态服务上。

## 3. m3u8 和 MP4 的区别

| 对比     | MP4         | m3u8 / HLS                            |
| ------ | ----------- | ------------------------------------- |
| 本质     | 单个视频文件      | 播放清单 + 多个视频分片                         |
| 文件形式   | `video.mp4` | `index.m3u8 + xxx.ts/m4s`             |
| 适合场景   | 短视频、普通点播    | 长视频、直播、自适应清晰度                         |
| 播放复杂度  | 简单          | 需要 HLS 支持或 JS 播放库                     |
| CDN 缓存 | 缓存一个大文件     | 缓存很多小片段                               |
| 清晰度切换  | 通常要手动换地址    | 可以自动切换                                |
| 首屏加载   | 简单视频很快      | 配置合理时长视频体验更好                          |
| 直播     | 不适合         | 很适合                                   |
| 浏览器兼容  | 原生支持更广      | Safari 原生好，Chrome/Firefox 通常需要 hls.js |

所以不是说 m3u8 一定比 MP4 好。

简单判断：

```text
短视频、几十秒、一两分钟：MP4 更简单
长视频、课程、讲书、直播、弱网环境：m3u8 / HLS 更合适
需要自动清晰度切换：m3u8 / HLS 更合适
```

## 4. web 端 video 能不能直接播放 m3u8？

分浏览器。

### Safari / iOS Safari

通常可以直接：

```html
<video
  src="https://cdn.xxx.com/video/master.m3u8"
  controls
  playsinline
></video>
```

Safari 对 HLS 原生支持比较好。

---

### Chrome / Edge / Firefox

大多数情况下，不能直接把 m3u8 丢给 `<video>` 播，需要用 JS 库解析 m3u8，再通过 MSE 喂给 video 播放。

MDN 说明 Media Source Extensions，也就是 MSE，允许 JavaScript 动态构建 media stream 并交给 audio/video 播放；现代浏览器常以 H.264、AAC、MP4 容器作为常见基线。([MDN Web Docs][3])

所以常见方案是：

```text
浏览器 video 标签
    ↑
hls.js / video.js
    ↑
m3u8 + ts/m4s 分片
```

## 5. 最常用 npm 包：hls.js

如果你只是要在 Web 播 m3u8，最直接用：

```bash
pnpm add hls.js
```

HTML：

```html
<video
  id="video"
  controls
  playsinline
  preload="metadata"
  poster="/poster.jpg"
></video>
```

JS / TS：

```ts
import Hls from 'hls.js'

const video = document.querySelector<HTMLVideoElement>('#video')!
const url = 'https://cdn.xxx.com/video/master.m3u8'

if (video.canPlayType('application/vnd.apple.mpegurl')) {
  // Safari / iOS Safari：原生 HLS
  video.src = url
} else if (Hls.isSupported()) {
  // Chrome / Edge / Firefox：hls.js + MSE
  const hls = new Hls({
    enableWorker: true,
  })

  hls.loadSource(url)
  hls.attachMedia(video)
} else {
  console.error('当前浏览器不支持 HLS 播放')
}
```

`hls.js` 官方说明它是一个实现 HLS 客户端的 JavaScript 库，依赖 HTML5 video 和 MediaSource Extensions 播放；它的兼容性也要求浏览器支持 MSE。([nochev.github.io][4])

`canPlayType()` 是 HTMLMediaElement 的方法，用来判断浏览器是否可能播放某种媒体类型。([MDN Web Docs][5])

## 6. Vue 里怎么写？

你如果是 Vue3：

```vue
<template>
  <video
    ref="videoRef"
    controls
    playsinline
    preload="metadata"
    class="video"
  ></video>
</template>

<script setup lang="ts">
import Hls from 'hls.js'
import { onBeforeUnmount, onMounted, ref } from 'vue'

const videoRef = ref<HTMLVideoElement | null>(null)
const hlsRef = ref<Hls | null>(null)

const videoUrl = 'https://cdn.xxx.com/video/master.m3u8'

onMounted(() => {
  const video = videoRef.value
  if (!video) return

  if (video.canPlayType('application/vnd.apple.mpegurl')) {
    video.src = videoUrl
    return
  }

  if (Hls.isSupported()) {
    const hls = new Hls({
      enableWorker: true,
    })

    hls.loadSource(videoUrl)
    hls.attachMedia(video)

    hlsRef.value = hls
  }
})

onBeforeUnmount(() => {
  hlsRef.value?.destroy()
  hlsRef.value = null
})
</script>

<style scoped>
.video {
  width: 100%;
  height: auto;
  background: #000;
}
</style>
```

注意：组件销毁时一定要 `hls.destroy()`，否则可能有内存、网络请求、事件监听残留。

## 7. React 里怎么写？

```tsx
import Hls from 'hls.js'
import { useEffect, useRef } from 'react'

export function HlsVideo() {
  const videoRef = useRef<HTMLVideoElement | null>(null)

  useEffect(() => {
    const video = videoRef.current
    if (!video) return

    const url = 'https://cdn.xxx.com/video/master.m3u8'
    let hls: Hls | null = null

    if (video.canPlayType('application/vnd.apple.mpegurl')) {
      video.src = url
    } else if (Hls.isSupported()) {
      hls = new Hls({
        enableWorker: true,
      })

      hls.loadSource(url)
      hls.attachMedia(video)
    }

    return () => {
      hls?.destroy()
    }
  }, [])

  return (
    <video
      ref={videoRef}
      controls
      playsInline
      preload="metadata"
      style={{ width: '100%', background: '#000' }}
    />
  )
}
```

```bash

const player = videojs('my-video', {

  sources: [
    {

```

HTML：

```html
<video
  id="my-video"
  class="video-js vjs-default-skin"

  playsinline

```

适合：

```text
业务播放器
课程视频

```

## 10. 服务端怎么生成 m3u8？

最常见用 FFmpeg。

### 单清晰度 HLS

```bash
ffmpeg -i input.mp4 \
  -c:v h264 \
  -c:a aac \
  -hls_time 6 \
  -hls_playlist_type vod \
  -hls_segment_filename "segment_%03d.ts" \
  index.m3u8
```

生成结果：

```text
index.m3u8
segment_000.ts
segment_001.ts
segment_002.ts
```

FFmpeg 官方文档说明，HLS muxer 会创建一个 playlist 文件和一个或多个 segment 文件；默认会为每个 segment 生成 `.ts` 文件。([FFmpeg][8])

`hls_time` 是目标分片时长，`hls_playlist_type vod` 会生成点播类型播放列表。([FFmpeg][8])

---

### 多清晰度 HLS

真实业务里一般不是只切一个清晰度，而是生成多档：

```text
360p
720p
1080p
master.m3u8
```

大致结构：

```text
video/
  master.m3u8
  360p/
    index.m3u8
    segment_000.ts
  720p/
    index.m3u8
    segment_000.ts
  1080p/
    index.m3u8
    segment_000.ts
```

前端只播放：

```text
https://cdn.xxx.com/video/master.m3u8
```

播放器自己选择清晰度。

## 11. Web 端配合时要注意什么？

### CDN / 服务端 MIME 类型

建议配置：

```text
.m3u8  -> application/vnd.apple.mpegurl
.ts    -> video/mp2t
.m4s   -> video/iso.segment 或 application/octet-stream
.mp4   -> video/mp4
```

IANA 注册的 m3u8 MIME 类型是 `application/vnd.apple.mpegurl`。([互联网分配号码管理局][9])

---

### CORS

如果你的页面域名是：

```text
https://www.xxx.com
```

视频在：

```text
https://cdn.xxx.com
```

那 CDN 要允许跨域：

```http
Access-Control-Allow-Origin: *
```

或者允许你的业务域名。

否则 hls.js 拉 m3u8 或 ts 分片可能失败。

---

### video 标签建议配置

```html
<video
  controls
  playsinline
  preload="metadata"
  poster="/poster.jpg"
></video>
```

常用属性：

| 属性                   | 作用                 |
| -------------------- | ------------------ |
| `controls`           | 显示播放控件             |
| `playsinline`        | 移动端内联播放，避免强制全屏     |
| `preload="metadata"` | 只预加载元信息，别一上来拉太多    |
| `poster`             | 视频封面               |
| `muted`              | 静音，常用于自动播放         |
| `autoplay`           | 自动播放，但通常需要配合 muted |

MDN 对 `<video>` 的属性有完整说明，包括 `autoplay`、`controls`、`muted`、`playsinline`、`poster`、`preload` 等。([MDN Web Docs][10])

---

### 自动播放限制

现代浏览器通常限制有声音的视频自动播放。
如果你要自动播放，一般要：

```html
<video autoplay muted playsinline></video>
```

否则很多浏览器会拦截。

## 12. 在 uniapp / 小程序里呢？

如果是 **Web H5**，用上面这些方案。

如果是 **微信小程序 video 组件**，一般直接给 `src`：

```vue
<video
  src="https://cdn.xxx.com/video/master.m3u8"
  controls
></video>
```

但小程序里重点要注意：

```text
1. 视频域名必须配置到微信小程序后台合法域名
2. m3u8 和 ts 分片都必须走 HTTPS
3. CDN 必须允许访问所有分片
4. 编码最好用 H.264 + AAC
5. 不要用 hvc1 / HEVC 作为默认编码，否则部分设备黑屏有声音
```

你之前遇到“长视频黑屏有声音、编码是 hvc1”，这个就很像 **视频编码兼容问题**。
m3u8 只是分发方式，不能自动解决编码兼容。你还是要把视频转成更通用的：

```text
H.264 视频 + AAC 音频
```

## 13. 什么时候选 MP4，什么时候选 m3u8？

### 用 MP4

```text
视频很短
不需要直播
不需要自动清晰度切换
不想引入播放器库
兼容和开发成本优先
```

比如：

```text
首页短视频
宣传视频
几秒到几分钟的视频
简单预览视频
```

---

### 用 m3u8 / HLS

```text
视频比较长
用户网络环境复杂
需要边下边播更稳定
需要自动清晰度切换
需要直播
需要 CDN 分片缓存
需要更专业的视频播放体验
```

比如：

```text
课程视频
讲书音频/视频
长视频内容
直播
AI 生成任务回放
```
