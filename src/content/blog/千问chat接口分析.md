我帮你**完整拆解**这个 curl 请求：这是**通义千问（阿里千问）网页版的私有AI对话接口**，专门用于**AI文生图 + 对话交互**，我分三部分讲清楚：**接口是什么**、**用到的所有参数**、**关键注意事项**。

# 一、接口核心信息
1. **接口地址**
`POST https://chat2.qianwen.com/api/v1/chat/snap`
2. **接口作用**
千问网页版**私有核心接口**：发送用户对话请求，获取AI流式回复（当前请求是**AI生成图片**场景）
3. **请求类型**
POST + **SSE 事件流**（响应是实时流式输出）
4. **归属产品**
阿里云 · 通义千问（qianwen.com）网页版

# 二、全量参数拆解（还原所有参数+用途）
我把 curl 里的转义符全部清理，分 **URL 参数、请求头、请求体** 三部分说明

## 1）URL Query 参数（? 后面的公共参数）
全部是千问网页版**固定公共参数**，用于环境标识、防重放、签名：
| 参数名 | 值 | 作用 |
|--------|----|------|
| biz_id | ai_qwen | 业务ID（固定千问业务） |
| fe_version | 1.0.0 | 前端版本号 |
| chat_client | h5 | 客户端类型（H5网页） |
| device | pc | 设备类型（PC端） |
| fr | pc | 来源渠道 |
| pr | qwen | 产品标识（千问） |
| ut | UUID | 设备唯一ID |
| la | zh-CN | 语言 |
| tz | Asia/Shanghai | 时区 |
| wv / ve | 2.6.1 | 客户端版本 |
| nonce | 随机字符串 | 防重放攻击 |
| timestamp | 时间戳 | 请求时间，签名用 |

## 2）请求头 Header（分3类：通用头+鉴权头+业务头）
### ① 通用浏览器头（所有网页请求都带）
`accept、accept-language、user-agent、content-type、origin、referer`
作用：标识浏览器、请求格式、来源页面

### ② 阿里私有**鉴权/签名头**（核心，接口调用必验）
| 头名称 | 作用 |
|--------|------|
| clt-acs-caer/reqt | 阿里客户端安全协议参数 |
| clt-acs-request-params | 参与签名的URL参数列表 |
| **clt-acs-sign** | **请求签名（防篡改，动态生成，无法伪造）** |
| eo-clt-* 系列 | 设备标识、会话令牌、安全校验 |
| Cookie | **登录态核心**（包含登录票据、会话ID、用户标识） |

### ③ 千问**业务头**（对话上下文）
| 头名称 | 作用 |
|--------|------|
| x-chat-biz / x-chat-id | 对话ID，标识当前聊天窗口 |
| x-device-id | 设备唯一ID |
| x-platform | 平台标识（pc端） |
| x-wpk-* | 阿里内部链路追踪ID |

## 3）请求体 Body（核心业务参数，AI生成图片）
这是**真正发给AI的指令**，当前请求是**文生图**：
| 字段 | 作用 |
|------|------|
| req_id | 请求唯一ID |
| session_id | 会话ID（当前聊天窗口） |
| messages | 用户发送的消息（空内容，走biz_data指令） |
| model | Qwen（千问模型） |
| biz_data | **图片生成核心配置**：<br>模型=qwen2<br>尺寸=3:4<br>生成类型=AI绘画 |
| scene | chat（对话场景） |
| ai_tool_scene | zaodian_generate_image（文生图工具） |

# 三、关键结论（你最关心的）
## 1. 这个接口是干嘛的？
**阿里千问网页版 → AI文生图 + 对话的私有接口**
不是开放平台接口，**不对外提供调用**。

## 2. 能不能直接复制 curl 调用？
**不能！绝对不能！**
原因：
1. **强签名校验**：`clt-acs-sign` 是动态签名，绑定设备+时间+Cookie，过期/篡改直接失败
2. **登录态绑定**：Cookie 里的 `tongyi_sso_ticket` 是你的**个人登录凭证**，泄露会被盗号
3. **私有接口**：阿里随时会修改参数、路径、签名规则

## 3. 想调用千问AI，正确方式？
用**阿里云官方开放 API**（通义千问 SDK/OpenAPI），不要抓网页接口。
